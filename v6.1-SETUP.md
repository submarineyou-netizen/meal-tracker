# v6.1 セットアップ手順

## ⚠️ 重要：Firebaseの設定が必要です

このバージョンから Google アカウントでのログインとクラウド同期機能が追加されました。
使用するには Firebase プロジェクトの作成と設定が必要です。

---

## 📋 準備するもの

1. Googleアカウント（既にお持ちのものでOK）
2. PC（Firebaseの設定作業用）

---

## 🔧 セットアップ手順

### ステップ1: Firebaseプロジェクトを作成

同梱の `FIREBASE_SETUP.md` を見ながら、Firebaseコンソールでプロジェクトを作成してください。

所要時間: 約10分

### ステップ2: 設定値をコピー

Firebaseコンソールで取得した設定値（6つ）をメモします：

```javascript
apiKey: "AIza..."
authDomain: "meal-tracker-xxxxx.firebaseapp.com"
projectId: "meal-tracker-xxxxx"
storageBucket: "meal-tracker-xxxxx.appspot.com"
messagingSenderId: "123456789"
appId: "1:123456789:web:xxxxx"
```

### ステップ3: ソースコードに設定値を反映

⚠️ **この作業が必要です**

ZIPを解凍後、以下のファイルを編集します：

#### ファイルパス
```
meal-tracker-v6.1/assets/index-CtlIpiUn.js
```

#### 編集方法

1. `index-CtlIpiUn.js` をテキストエディタで開く
2. `Ctrl+F`（Mac: `Cmd+F`）で検索
3. `YOUR_API_KEY` を検索
4. 見つかった箇所の設定値を、実際の値に置き換える

**置き換え前：**
```javascript
apiKey:"YOUR_API_KEY",authDomain:"YOUR_PROJECT_ID.firebaseapp.com",...
```

**置き換え後（例）：**
```javascript
apiKey:"AIzaSyAbc123...",authDomain:"meal-tracker-12345.firebaseapp.com",...
```

5. 以下の6つすべてを置き換え：
   - `YOUR_API_KEY` → 実際のapiKey
   - `YOUR_PROJECT_ID.firebaseapp.com` → 実際のauthDomain
   - `YOUR_PROJECT_ID` → 実際のprojectId（2箇所）
   - `YOUR_PROJECT_ID.appspot.com` → 実際のstorageBucket
   - `YOUR_MESSAGING_SENDER_ID` → 実際のmessagingSenderId
   - `YOUR_APP_ID` → 実際のappId

6. 保存

### ステップ4: GitHubにアップロード

編集した `meal-tracker-v6.1` フォルダをGitHubにアップロードします。

### ステップ5: 動作確認

1. サイトを開く
2. Googleログイン画面が表示される
3. 「Googleでログイン」をクリック
4. Googleアカウントを選択
5. ログイン成功 → アプリのホーム画面が表示される

---

## ❓ トラブルシューティング

### ログインボタンを押してもエラーが出る

→ Firebase の設定値が正しく入っていない可能性があります
→ `index-CtlIpiUn.js` の設定値を再確認してください

### ログイン画面が表示されない

→ ブラウザのキャッシュをクリアしてください

---
